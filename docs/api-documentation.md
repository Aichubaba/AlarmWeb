# API документация

Базовый URL: `http://localhost:5159` / `https://localhost:7086`

Формат ответа: `application/json`

В среде разработки доступен Swagger UI:
```
https://localhost:7086/swagger
```

---

## Аутентификация

### `POST /api/auth/login`

Вход в систему. Создаёт cookie-сессию с аутентификацией.

**Параметры формы (x-www-form-urlencoded):**

| Параметр   | Тип    | Обязательный | Описание              |
|------------|--------|--------------|-----------------------|
| `username` | string | да           | Имя пользователя      |
| `password` | string | да           | Пароль                |

**Пример запроса:**
```
POST /api/auth/login
Content-Type: application/x-www-form-urlencoded

username=admin&password=admin123
```

**Успешный ответ:** `302 Found` — перенаправление на `/`

**Неуспешный ответ:** `302 Found` — перенаправление на `/login?error=1`

**Доступные учётные данные:**

| Логин | Пароль   | Роль  |
|-------|----------|-------|
| admin | admin123 | admin |
| user  | user123  | user  |

---

### `GET /api/auth/logout`

Выход из системы. Удаляет cookie-сессию.

**Пример запроса:**
```
GET /api/auth/logout
```

**Ответ:** `302 Found` — перенаправление на `/login`

## Алармы (требуется авторизация)

### `POST /api/alarms/receive`

Приём алармов от внешних систем.

**Заголовки:**
```
Authorization: <cookie-сессия>
Content-Type: application/json
```

**Тело запроса:**

| Поле        | Тип    | Обязательный | Описание                |
|-------------|--------|--------------|-------------------------|
| `source`    | string | да           | Источник аларма         |
| `message`   | string | да           | Текст сообщения         |
| `timestamp` | string | да           | Временная метка (ISO 8601) |

**Пример запроса:**
```json
POST /api/alarms/receive
Content-Type: application/json

{
  "source": "sensor-01",
  "message": "Temperature exceeds threshold",
  "timestamp": "2026-05-11T10:30:00Z"
}
```

**Успешный ответ:** `200 OK`
```json
{
  "status": "принято"
}
```

> **Примечание:** В текущей версии алармы только логируются в консоль. Интеграция с базой данных будет добавлена в следующих релизах.

## Мониторинг

### `GET /health`

Проверка работоспособности приложения.

**Пример запроса:**
```
GET /health
```

**Ответ:** `200 OK` — приложение работает

---

### `GET /metrics`

Заглушка для Prometheus-метрик. В текущей версии возвращает текстовое сообщение.

**Пример запроса:**
```
GET /metrics
```

**Ответ:** `200 OK`
```
# Prometheus metrics will be here
```

---

## Коды ответов

| Код | Описание                                           |
|-----|----------------------------------------------------|
| 200 | Успешный ответ                                     |
| 302 | Перенаправление (используется при логине/логауте)  |
| 401 | Не авторизован                                     |
| 403 | Доступ запрещён                                    |
| 404 | Страница не найдена                                |
| 500 | Внутренняя ошибка сервера                          |
